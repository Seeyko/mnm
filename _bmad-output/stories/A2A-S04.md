# A2A-S04 — Connecteurs MCP (Model Context Protocol)

> **Epic** : A2A — Agent-to-Agent + Permissions (Noyau D)
> **Sprint** : Batch 14
> **Assignation** : Cofondateur
> **Effort** : M (3 SP)
> **Bloqué par** : (aucune dépendance stricte — P2)
> **Débloque** : (rien — dernière story A2A)
> **Priorité** : P2

---

## Contexte

MnM orchestre des agents AI qui communiquent via le A2A Bus (A2A-S01). Le Model Context Protocol (MCP) est un standard ouvert (Anthropic) qui permet aux agents d'accéder à des outils et services externes via des "connecteurs" (serveurs MCP). A2A-S04 ajoute un **MCP Connector Registry** dans MnM : les admins enregistrent des serveurs MCP (URL + transport + auth), et les agents peuvent lister les connecteurs disponibles, invoquer des outils MCP, et recevoir les résultats — le tout audité et contrôlé par les permissions existantes.

**Ce que MnM ne fait PAS** :
- MnM ne lance pas de serveurs MCP — il s'y connecte en tant que client
- MnM ne remplace pas le bus A2A — les connecteurs MCP sont une source d'outils externe complémentaire
- L'invocation réelle des serveurs MCP est simulée/stubbed pour l'instant (P2) — la priorité est le registry, le CRUD, les types, les routes, et l'audit

## Acceptance Criteria (Given/When/Then)

### AC1 — Enregistrement d'un connecteur MCP
**Given** un admin/manager
**When** il envoie POST /companies/:companyId/a2a/mcp-connectors avec name, url, transport, authType
**Then** le connecteur est créé dans la base avec statut "active", un audit event `a2a.mcp_connector_created` est émis

### AC2 — Liste des connecteurs MCP
**Given** un utilisateur avec permission agents:create
**When** il envoie GET /companies/:companyId/a2a/mcp-connectors
**Then** il reçoit la liste des connecteurs MCP de la company, filtrables par status et transport

### AC3 — Détail d'un connecteur MCP
**Given** un connecteur MCP existant
**When** GET /companies/:companyId/a2a/mcp-connectors/:connectorId
**Then** il reçoit les détails complets du connecteur (name, url, transport, authType, toolCount, status, etc.)

### AC4 — Mise à jour d'un connecteur MCP
**Given** un admin/manager
**When** il envoie PUT /companies/:companyId/a2a/mcp-connectors/:connectorId avec des champs modifiés
**Then** le connecteur est mis à jour, un audit event `a2a.mcp_connector_updated` est émis

### AC5 — Suppression d'un connecteur MCP
**Given** un admin/manager
**When** il envoie DELETE /companies/:companyId/a2a/mcp-connectors/:connectorId
**Then** le connecteur est supprimé, un audit event `a2a.mcp_connector_deleted` est émis

### AC6 — Test de connectivité MCP
**Given** un connecteur MCP enregistré
**When** l'admin envoie POST /companies/:companyId/a2a/mcp-connectors/:connectorId/test
**Then** le service tente de se connecter au serveur MCP, retourne { reachable: boolean, latencyMs, toolCount, error? }, un audit event `a2a.mcp_connector_tested` est émis

### AC7 — Liste des outils MCP d'un connecteur
**Given** un connecteur MCP actif
**When** GET /companies/:companyId/a2a/mcp-connectors/:connectorId/tools
**Then** il reçoit la liste des outils exposés par ce serveur MCP (name, description, inputSchema)

### AC8 — Invocation d'un outil MCP
**Given** un connecteur MCP actif avec l'outil "search_documents"
**When** POST /companies/:companyId/a2a/mcp-connectors/:connectorId/tools/:toolName/invoke avec inputArgs
**Then** le résultat de l'outil est retourné, un audit event `a2a.mcp_tool_invoked` est émis avec toolName, connectorId, durationMs

### AC9 — Statistiques MCP par company
**Given** un admin
**When** GET /companies/:companyId/a2a/mcp-connectors/stats
**Then** il reçoit { totalConnectors, activeCount, inactiveCount, totalToolInvocations, averageLatencyMs }

### AC10 — Désactivation d'un connecteur
**Given** un connecteur MCP actif
**When** PUT avec status: "inactive"
**Then** le connecteur est désactivé, aucun agent ne peut l'invoquer, un audit event `a2a.mcp_connector_updated` est émis avec metadata { statusChange: "active→inactive" }

### AC11 — LiveEvent notifications
**Given** un connecteur MCP créé/testé/outil invoqué
**When** l'action se produit
**Then** un LiveEvent `a2a.mcp_connector_changed` est publié sur le companyId

### AC12 — Types et validators partagés
**Given** le package shared
**When** les types MCP sont définis
**Then** McpConnector, McpTool, McpToolInvocationResult sont exportés depuis @mnm/shared, avec validators Zod pour create/update/invoke

## Deliverables

### D1 — Types partagés (`packages/shared/src/types/a2a.ts`)
Ajouter en fin de fichier :

```typescript
// A2A-S04: MCP Connector types
export const MCP_TRANSPORT_TYPES = ["stdio", "sse", "streamable-http"] as const;
export type McpTransportType = (typeof MCP_TRANSPORT_TYPES)[number];

export const MCP_AUTH_TYPES = ["none", "bearer", "api_key", "oauth2"] as const;
export type McpAuthType = (typeof MCP_AUTH_TYPES)[number];

export const MCP_CONNECTOR_STATUSES = ["active", "inactive", "error"] as const;
export type McpConnectorStatus = (typeof MCP_CONNECTOR_STATUSES)[number];

export interface McpConnector {
  id: string;
  companyId: string;
  name: string;
  description: string | null;
  url: string;
  transport: McpTransportType;
  authType: McpAuthType;
  authConfig: Record<string, unknown> | null;
  status: McpConnectorStatus;
  toolCount: number;
  lastTestedAt: string | null;
  lastTestResult: McpConnectorTestResult | null;
  createdAt: string;
  updatedAt: string;
}

export interface McpTool {
  name: string;
  description: string | null;
  inputSchema: Record<string, unknown>;
}

export interface McpToolInvocationResult {
  connectorId: string;
  toolName: string;
  success: boolean;
  result: unknown;
  error: string | null;
  durationMs: number;
  invokedAt: string;
}

export interface McpConnectorTestResult {
  reachable: boolean;
  latencyMs: number;
  toolCount: number;
  error: string | null;
  testedAt: string;
}

export interface McpConnectorStats {
  totalConnectors: number;
  activeCount: number;
  inactiveCount: number;
  errorCount: number;
  totalToolInvocations: number;
  averageLatencyMs: number | null;
}

export interface McpConnectorFilters {
  status?: McpConnectorStatus;
  transport?: McpTransportType;
  limit?: number;
  offset?: number;
}
```

## Notes d'implémentation

- **Storage** : Les connecteurs MCP sont stockés en mémoire (Map) dans le service — pas de nouvelle table DB. C'est un choix P2 : la persistance DB sera ajoutée si/quand MCP devient P0.
- **Stubbed invocation** : testConnector et invokeTool retournent des résultats simulés. L'intégration réelle avec le SDK MCP est hors scope.
- **Sécurité** : Les authConfig (tokens, API keys) ne doivent JAMAIS apparaître dans les audit logs ou les réponses API (masquage).
- **Pattern** : Suit le même service factory pattern que a2a-bus.ts et a2a-permissions.ts.
